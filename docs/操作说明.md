## 1. 安装与启动

### 安装版

双击 `.exe` 安装包，按向导完成安装，从开始菜单启动「视频批量处理器」。

### 开发模式

```powershell
npm run tauri dev
```

> **注意：** 首次启动时 Windows 可能弹出安全提示，点击「更多信息」→「仍要运行」即可。

## 2. 主界面介绍

界面分为以下几个区域：

| 区域 | 功能 |
|------|------|
| 文件夹设置 | 选择输入和输出文件夹，设置文件扩展名 |
| 处理选项 | 设置并行数量和画质 |
| 多机分片 | 多台机器并行时使用 |
| 进度 | 显示处理进度条和文件统计（等待/完成/失败） |
| 日志 | 实时显示处理详情 |
| 底部操作栏 | 开始处理、停止、查询状态 |

## 3. 基本使用流程

### 第一步：选择输入文件夹

点击「输入文件夹」右侧的「浏览」按钮，选择存放源视频的文件夹。

### 第二步：选择输出文件夹

点击「输出文件夹」右侧的「浏览」按钮，选择 MP4 文件保存位置。

> **注意：** 输出文件夹可以为空文件夹，程序会自动创建。

### 第三步：确认文件扩展名

默认：`.ts,.mp4,.mkv,.avi,.mov`

- 如果只需要处理某一种格式，可以只保留对应的后缀，例如只写 `.ts`
- 多种格式用英文逗号分隔

### 第四步：选择画质

根据需要选择画质档位（详见[画质选择说明](#4-画质选择说明)）。

### 第五步：点击「开始处理」

点击后程序开始扫描文件并处理，日志区域会实时显示进度。

### 中断与恢复

- 点击「停止」按钮可随时中断
- 已完成的文件**不会重复处理**，下次启动后直接继续

## 4. 画质选择说明

| 选项 | 编码方式 | 速度 | 画质 | 适用场景 |
|------|----------|------|------|----------|
| 高画质 | CPU（libx264） | 慢 | 最佳 | 存档、对质量要求高 |
| 中画质 | GPU（NVENC 高质量） | 快 | 好 | 日常使用，有 NVIDIA 显卡 |
| 低画质 | GPU（NVENC 极速） | 极快 | 一般 | 快速预览、批量测试 |

**无 NVIDIA 显卡时：** 中画质和低画质选项会显示「— 无 GPU」且不可选，自动回退为高画质（CPU）模式。

## 5. 多机并行处理

当有多台电脑需要共同处理同一批视频时，可以使用「多机分片」功能。

### 设置步骤

1. 勾选「多机分片（可选）」复选框
2. 设置「本机编号」（从 0 开始）
3. 设置「总机器数量」（所有机器填相同的数字）
4. 每台机器设置**各自独立**的输出文件夹

**示例（3台机器处理30个文件）：**

| 机器 | 本机编号 | 总机器数量 | 处理文件 |
|------|----------|------------|----------|
| 机器A | 0 | 3 | 第1、4、7、10...个 |
| 机器B | 1 | 3 | 第2、5、8、11...个 |
| 机器C | 2 | 3 | 第3、6、9、12...个 |

## 6. 查看处理状态

点击「查询状态」按钮，日志区域会显示当前进度统计：

- **等待**：尚未处理的文件数
- **完成**：成功处理的文件数
- **失败**：处理出错的文件数（失败文件会列出文件名和原因）

失败的文件不会阻止其他文件继续处理，可以在全部完成后单独检查失败原因。

## 8. 常见问题

**Q：启动后提示「Windows 已保护你的电脑」**
A：点击「更多信息」→「仍要运行」，这是 Windows 对未签名程序的正常提示。

**Q：启动时有短暂等待**
A：正常现象，程序启动时自动检测 GPU 是否可用。

**Q：中画质/低画质是灰色不能选**
A：当前电脑没有 NVIDIA 显卡或显卡驱动版本过低，请使用高画质（CPU）模式。

**Q：处理失败了怎么办**
A：点击「查询状态」查看失败文件列表，检查：①源文件是否损坏；②输出磁盘是否有足够空间。

**Q：中断后重新开始，会重复处理已完成的文件吗**
A：不会。已完成的文件会自动跳过，从断点继续。

**Q：多台机器处理完后，文件在哪里**
A：每台机器的输出文件在各自设置的「输出文件夹」中，汇总时将各机器的输出文件夹合并即可。

**Q：并行数量设置多少合适**
A：建议设置为电脑物理核心数的一半。例如 8 核电脑建议设置为 4。使用 GPU 模式时设置为 1~2 即可。

**Q：`npm run tauri dev` 启动失败**
A：检查：①Node.js >= 18；②Rust 已安装；③`cargo tauri --version` 有输出；④ffmpeg 在 PATH 或 `tools/ffmpeg/` 中。
